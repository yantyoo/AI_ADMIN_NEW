# IA_SPEC

## 1. 문서 목적
- 현재 구현된 관리자 서비스의 화면 구조와 사용자 흐름을 페이지 단위로 정의한다.
- 화면 배치, 접근 뎁스, 상태 전이, 메시지 위치를 명확히 해서 프론트와 기획이 같은 기준으로 작업할 수 있게 한다.
- 기능 상세와 정책은 `FEATURE_SPEC.md`를 따른다.

## 2. 뎁스 정의

| 구분 | 정의 |
|---|---|
| L1 | URL 기준 최상위 페이지 |
| L2 | 화면의 주요 영역 / 패널 단위 |
| L3 | 사용자가 직접 조작하는 입력, 버튼, 목록 항목, 상태 요소 |

## 3. 공통 레이아웃 원칙

| 영역 | 역할 | 주요 구성요소명 |
|---|---|---|
| G-01 LNB | 좌측 고정 내비게이션 | `Sidebar`, `sidebar__brand`, `sidebar__nav`, `sidebar__user` |
| G-02 상단 헤더 | 현재 페이지 제목 / 보조 정보 | `TopHeader` |
| G-03 본문 영역 | 페이지별 콘텐츠 표시 | `admin-shell__content` |
| G-04 모달 | 확인 / 등록 / 수정 / 삭제 | `modal-backdrop`, `modal`, `modal__header`, `modal__body`, `modal__footer` |

### 공통 배치 원칙
- LNB는 항상 고정한다.
- 본문만 스크롤된다.
- 목록과 상세는 분리된 패널로 구성한다.
- 모달은 페이지 위에 오버레이로 띄운다.
- 빈 상태와 에러 상태는 같은 영역에서 교체 표시한다.

### 공통 상태 원칙
- `loading`: 스켈레톤 또는 로딩 문구 표시
- `empty`: 데이터 없음 문구 표시
- `error`: 에러 문구와 재시도 버튼 표시
- `success`: 정상 렌더링
- `disabled`: 권한 없음 또는 동작 불가 상태

## 4. 페이지별 와이어프레임

### 4.1 로그인

#### L1
- `/login`

#### 와이어프레임
```text
┌────────────────────────────────────────────────────────────┐
│ auth-shell                                                │
│  └─ auth-card                                             │
│      ├─ auth-card__intro                                  │
│      │   ├─ 배지 / 타이틀                                 │
│      │   └─ 안내 문구                                     │
│      ├─ auth-form                                         │
│      │   ├─ auth-form__fields                             │
│      │   │   ├─ 아이디 입력                               │
│      │   │   ├─ 비밀번호 입력                             │
│      │   │   └─ 아이디 저장                               │
│      │   └─ auth-form__actions                            │
│      │       ├─ 로그인 버튼                               │
│      │       └─ 취소 버튼                                 │
│      └─ auth-form__feedback                               │
│          ├─ 에러 메시지                                   │
│          └─ 보조 안내 문구                                │
└────────────────────────────────────────────────────────────┘
```

#### L2
- A-01 인증 카드
- A-02 소개 영역
- A-03 입력 폼
- A-04 피드백 영역

#### L3
- A-03-01 아이디 입력
- A-03-02 비밀번호 입력
- A-03-03 아이디 저장
- A-03-04 로그인 버튼
- A-03-05 취소 버튼
- A-04-01 에러 메시지
- A-04-02 보조 안내 문구
- A-04-03 계정/비밀번호 오류
- A-04-04 권한 없음 오류
- A-04-05 비인가 계정 차단 안내
- A-04-06 OTP 잠금 오류

#### 상태 전이
- 입력 완료 -> XpERP 계정 인증 -> OTP 화면 이동
- 필수값 누락 -> 입력 영역 아래 에러 표시
- 계정/비밀번호 오류 -> 폼 하단 에러 표시
- 권한 없음/비인가 계정 -> 공통 에러 표시 후 입력 유지
- OTP 잠금 계정 -> 로그인 화면에서 차단 메시지 표시

#### 메시지 위치
- 입력 에러: 해당 필드 바로 아래
- 공통 에러: 폼 하단
- 권한/비인가 차단: 폼 하단 또는 카드 내 경고 영역

- B-01 인증 카드

#### L3
- B-03-01 OTP 입력
- B-03-02 실패 횟수 표시

- B-03-04 확인 버튼
- B-03-05 취소 버튼

- B-04-02 OTP 계정 잠금 메시지

#### 상태 전이
- OTP 제출 -> 검증 -> 인증 완료 -> 대시보드 진입
- 입력값 오류 -> 입력창 아래 에러 표시
- 실패 1~4회 -> 실패 횟수 갱신 + 안내 메시지 표시
- 실패 5회 -> 계정 잠금 + 잠금 안내 표시
- 잠금 계정 -> OTP 입력 차단
- 재전송 성공 -> 안내 메시지 표시

#### 메시지 위치
- 입력 오류: 입력창 아래
- 전송 결과: 폼 하단
- 실패 횟수: OTP 입력 바로 아래
- 잠금 안내: 폼 하단 또는 경고 배너

- 실패 -> 에러 상태와 재시도

#### 메시지 위치
- 목록 없음: 목록 영역 중앙
- 상세 없음: 상세 영역 중앙

#### 상태 전이
- 관리자 선택 -> 상세 갱신
- 권한 변경 -> 모달 처리 후 목록 / 상세 갱신
- 잠금 / 해제 -> 즉시 반영
- 관리자 추가 -> 후보 선택 후 사유 입력 완료 시 확인 가능

#### 메시지 위치
- 목록 없음: 목록 영역 중앙
- 권한 없음: 버튼 비활성화 또는 안내 문구
- 본인 계정 제한: 상세 영역 하단 안내 문구

#### 계정 모달 배치
- 관리자 추가 모달은 후보 검색과 사유 입력을 분리한다.
- 사유 입력은 확인 버튼 직전의 필수 입력으로 둔다.
- 잠금/해제 모달은 사유 없이는 확인 버튼을 활성화하지 않는다.

## 5. 공통 메시지 규칙

| 상황 | 표시 위치 | 문구 기준 |
|---|---|---|
| 성공 | 상단 토스트 | 작업 결과를 한 줄로 표시 |
| 실패 | 상단 토스트 또는 폼 하단 | 실패 사유를 짧게 표시 |
| 권한 없음 | 버튼 주변 또는 상세 영역 | 동작 불가 사유를 표시 |
| 중복 | 입력값 아래 | 중복 항목 기준을 함께 표시 |
| 데이터 없음 | 목록 / 상세 중앙 | 현재 조건에 데이터가 없다는 문구 |

## 6. 모달 배치 원칙
- 등록 모달은 입력 중심으로 구성한다.
- 수정 모달은 기존 값을 미리 채운다.
- 삭제 확인 모달은 제목 / 영향 범위 / 확인 버튼만 노출한다.
- 권한 변경은 별도 확인 단계를 둔다.
- 상세 모달은 외부 클릭으로 닫을 수 있다.
- 검증 오류는 모달 하단에 유지한다.

## 7. 문서 유지 원칙
- 페이지 구조는 현재 구현 기준으로 유지한다.
- 용어는 화면 문구와 동일하게 맞춘다.
- API 상세와 정책 상세는 `FEATURE_SPEC.md`에 둔다.
