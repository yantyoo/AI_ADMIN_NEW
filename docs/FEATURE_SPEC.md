# FEATURE_SPEC

## 1. 문서 목적
- 이 문서는 현재 구현된 기능을 백엔드 개발자가 바로 개발 가능한 수준으로 정리한다.
- 기능 번호(`FS-xxx`)를 유지해 기능 추적과 QA 기준을 맞춘다.
- 추정은 금지하고, 구현되지 않은 항목은 `OPEN_QUESTIONS`로 분리한다.
- 화면 배치와 IA는 `docs/IA_SPEC.md`를 따른다.

## 2. 공통 규칙

### 2.1 상태값

| 상태 | 의미 |
|---|---|
| `loading` | 데이터 조회 또는 저장 요청 중 |
| `empty` | 조회 결과 없음 |
| `error` | 조회/저장 실패 또는 검증 오류 |
| `success` | 조회/저장 성공 |
| `disabled` | 권한 없음, 입력 미완료, 중복 요청 차단 |

| 항목 | 기준 |

| 검색 방식 | 부분 일치 기반, 일부 화면은 유사도 정렬 포함 |

| 다중 정렬 | 현재 미사용 |

## 3. 기능 정의

### 3.1 인증

| FS ID | 기능명 | 설명 | 입력값 | 상태 | 결과 | 정책 |
|---|---|---|---|---|---|---|
| FS-001 | 로그인 입력 | 아이디를 입력한다 | `userId` | `default`, `error` | 다음 단계 진입 | 최대 10자, 영문/숫자 조합, 공백만 입력 불가 |

| FS-004 | OTP 입력 | OTP를 입력한다 | `otp` | `default`, `error` | 대시보드 진입 | 6자리 숫자 입력, 공백만 입력 불가 |

| FS-006 | 인증 상태 유지 | sessionStorage/localStorage에 인증 상태를 보존한다 | 없음 | `authenticated` | 재진입 허용 | 아이디는 LocalStorage 저장, 비밀번호는 저장 금지, 새로고침/재진입 시 인증 유지 |
| FS-007 | 인증 해제 | 로그아웃 시 인증 상태를 제거한다 | 없음 | `default` | `/login` 이동 | 로그아웃 확인 후 세션 인증값 제거, 중복 로그인 시 이전 세션 종료 |

#### 인증 정책
- 인증 원천은 XpERP 계정 인증이다.
- 로그인 허용 대상은 MASTER, OPERATOR로 제한한다.
- 아이디/비밀번호 인증 성공 후 OTP 2차 인증을 수행한다.
- 계정/비밀번호 오류와 권한 없음 오류는 분리해서 안내한다.
- OTP 실패 횟수는 비밀번호 실패와 분리해서 누적한다.
- OTP 5회 실패 시 계정을 잠근다.
- 잠금 해제는 MASTER만 가능하며, 사유 기록과 로그 저장이 필요하다.
- 잠금 해제 후 동일 계정이 다시 OTP 5회 실패하면 즉시 재잠금한다.

#### 로그인 메시지 정책

| 상황 | 메시지 |
|---|---|
| 계정/비밀번호 오류 | `아이디 또는 비밀번호가 올바르지 않습니다.` |
| 서버 연결 실패 | `서버 연결 오류, 잠시 후 다시 시도해주세요.` |
| 권한 없음 | `권한이 없는 사용자입니다. 관리자에게 권한을 요청해 주세요.` |
| 비인가 계정 | `MASTER, OPERATOR만 로그인할 수 있습니다.` |

#### OTP 정책

| 항목 | 기준 |
|---|---|
| 입력 규칙 | 6자리 숫자만 허용 |
| 버튼 활성화 | 6자리 입력 완료 시 활성화 |
| 실패 카운트 | 로그인 1차 인증 성공 이후 OTP 실패만 별도 집계 |
| 실패 안내 | 현재 실패 횟수를 함께 표시한다. 예: `2/5` |
| 잠금 안내 | `OTP 오류로 잠금된 아이디입니다. 관리자에게 문의하세요.` |
| 실패 메시지 | `OTP 5회 실패로 아이디가 잠겼습니다. 관리자에게 문의하세요.` |

#### 세션 정책

| 항목 | 기준 |
|---|---|
| 인증 유지 | 세션 인증값을 저장해 새로고침/재진입 시 유지한다 |
| 아이디 저장 | 아이디는 LocalStorage에 저장하고, 비밀번호는 저장하지 않는다 |
| 중복 로그인 | 동일 계정 중복 로그인은 허용하지 않으며, 이전 세션을 종료한다 |
| 로그아웃 | 인증 상태와 세션 정보를 제거하고 로그인 화면으로 이동한다 |

- KPI 카드는 값과 비교 기준만 노출하고 카드 하단 설명은 노출하지 않는다.
- 에러 상태는 메인 KPI/차트 영역을 대체하며, 재시도 버튼으로 정상 상태로 복귀한다.

#### 콘텐츠 정책

- 문서 검색은 문서명과 저장 경로를 기준으로 부분 일치 검색한다.
- 문서 검색은 2자 이상 입력을 권장하며, 실제 적용은 검색 버튼 기준으로 수행한다.
- 문서 유형은 `MANUAL` / `FAQ`만 사용한다.
- 업로드 편집 모달은 기존 값을 미리 채운다.
- 업로드 파일은 `.pdf`, `.docx`, `.txt`, `.md`만 허용한다.
- 문서 삭제 후에는 다음 문서를 자동 선택하고, 없으면 상세를 비운다.
- `FAILED` 상태는 업로드 실패 또는 검증 실패 문서에 사용한다.
- 다운로드는 현재 UI상 메시지 트리거이며, 실 파일 전송은 백엔드 계약이 필요하다.

#### 답변 정책

- 질문 검색은 정규화 후 부분 일치와 유사도 점수를 함께 사용한다.
- 질문 검색은 2자 이상 입력을 권장한다.
- 검색어가 있으면 관련도 우선, 없으면 최신순으로 정렬한다.
- 상태 필터 변경 시 페이지는 1페이지로 초기화한다.
- 목록은 10개 단위로 나누고, 페이지 이동은 상세 선택 상태와 분리한다.
- 질문과 답변은 각각 500자, 2000자까지 허용한다.
- 질문/답변 입력은 모달 하단 카운터로 길이를 노출한다.
- 중복 판정은 정규화, 포함 여부, Levenshtein 유사도로 판단한다.
- 등록/수정 후에는 목록과 상세를 즉시 갱신한다.
- 삭제 후에는 다음 항목을 자동 선택하고, 없으면 상세를 비운다.
- 상태 전환은 상세 영역에서 즉시 반영한다.
- 답변 요약은 목록에서 축약 노출한다.

#### 중복 판정 기준

| 기준 | 내용 |
|---|---|
| 정규화 | 공백 제거, 소문자화, 특수문자 제거 후 비교 |
| 유사도 | Levenshtein + 포함 여부 점수 |
| 임계값 | `0.85` 이상이면 중복으로 판단 |
| 예외 | 수정 시 자기 자신은 비교 대상에서 제외 |

#### 상태 전이

| 현재 상태 | 이벤트 | 다음 상태 |
|---|---|---|
| `ACTIVE` | 비활성화 | `INACTIVE` |
| `INACTIVE` | 활성화 | `ACTIVE` |
| `ACTIVE`/`INACTIVE` | 수정 | 현재 상태 유지 가능 |
| `ACTIVE`/`INACTIVE` | 삭제 | 제거 |

#### 상태 전이 판단
- 현재 구현 기준으로 `처리/미처리` 같은 별도 운영 상태는 없다.
- 피드백은 반응값만 있으며, 관리자는 상세 조회만 수행한다.

#### 피드백 정책

- 기간은 `일간`, `주간`, `월간` 3종만 사용한다.
- 필터는 `ALL`, `POSITIVE`, `NEGATIVE` 3종만 사용한다.
- 목록 행 클릭 또는 `상세` 버튼으로 상세를 연다.
- 부정 사유는 `NEGATIVE`이면서 사유가 존재할 때만 노출한다.
- 상세 영역은 사용자/챗봇 대화 버블과 상태 정보를 함께 보여준다.
- 전체 피드백 KPI는 노출하지 않는다.
- 긍정 반응 표기는 `만족해요`로 노출한다.
- 반응 필터 변경 시 목록만 재계산하고 상세 선택은 유지한다.
- 데이터가 없을 때는 목록 중앙에 빈 상태를 보여준다.
