# CLAUDE MD

## 1. 기준
- 기능 정의는 `FEATURE_SPEC.md`, 화면 구조는 `IA_SPEC.md`를 기준으로 한다.
- 현재 코드는 Vite + React 구조를 기준으로 본다.
- 문서와 화면 구조는 기존 레이아웃과 상태값을 최대한 유지한다.

## 2. 구조

| 구분 | 책임 |
|---|---|
| `src/*` | 앱 엔트리와 화면 조합 |
| `src/features/*` | 기능별 화면, 상태, 모달 |
| `src/components/*` | 공통 UI 컴포넌트 |
| `src/api/*` | mock 데이터 접근 |
| `src/types/*` | 공통 타입 |

## 3. 작업 원칙
- 구조 변경은 가능하지만 기능 디자인은 유지한다.
- 리스트/상세/모달 패턴은 바꾸지 않는다.
- 역할 분기와 상태값은 유지한다.
- 새 디자인 언어는 만들지 않는다.
